# specs/ の運用方法

機能単位の仕様を置くディレクトリ。1機能 = 1フォルダ(連番 + 機能名)。

```
specs/
└── 001-rss-collector/
    ├── spec.md    # 要件・受け入れ条件・スコープ外(人間が承認してからコミット)
    └── tasks.md   # 実装タスクの分解(実装フェーズでClaude Codeが生成)
```

## 1機能あたりのループ

1. **仕様の壁打ち**: Plan Mode(Shift+Tab ×2)で spec.md の草案を作らせる
2. **承認**: 人間が spec.md をレビューして修正させ、納得したらコミット
3. **テスト先行**: 受け入れ条件を vitest のテストケースに落とさせる
4. **実装**: テストが通るように実装させ、受け入れ条件との対応表で確認
5. 完了した spec.md の冒頭に `status: done` を追記

## spec.md のテンプレート

```markdown
# NNN: 機能名

- status: draft | approved | done

## 目的(なぜ作るか)

## 要件(何ができればよいか)

## 受け入れ条件

- Given(前提)/ When(操作)/ Then(期待結果)の形式で列挙

## スコープ外(今回やらないこと)

## 関連

- docs/DESIGN.md の該当セクション
```
